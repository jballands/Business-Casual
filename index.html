Why Business Casual

If there are two things that you need to know about web development, it's these: 1) today's web frameworks have a ton of dependencies, and 2) web developers like to make even the simpliest ideas as confusing as possible. The result is that, while you want to make a beautiful website, you're stuck looking at something like this.

Whether you're familar with web development or not, you'll probably download vast amounts of software just so you can make your personal website or portfolio. What you really need is a framework that not only looks good, but is also straightforward, honest, and contains very few dependencies.

Lo and behold, Business Casual: the framework that let's you that have your cake and eat it too.

Design Goals

In general, Business Casual has 3 design goals:

Be Structured

I wanted a framework that had a definite sense of structure and order right out of the box. With Business Casual, all the annoying fiddling and tweeking from standard web programming is handled by the framework, so you can focus more on what matters most: your content.

Work Fast

Business Casual has been trim and cut to be blazingly fast. You'll only find pure Javascript and LESS code in Business Casual. Better yet, the code is still readable.

Stay Beautiful

Nobody trusts an ugly website. That's why Business Casual stays pretty with tasteful fonts, colors, and design elements that helps the overall design of your website maintain a clean and professional look.

In general, Business Casual is divided into two parts: the static part and the dynamic part.

Static

Static features of Business Casual include:
  • Flexors
  • Standard navigation bars
  • Images
  • Tables
  • Wells
  • Buttons
  • Customization

Dynamic

Dynamic features of Business Casual include:
  • Sticky/magnetic navigation bars
  • Heroes
  • Glassboxes
  • Parallax

It's usually easier to include both the static and dynamic parts of Business Casual in your web page, but including only static parts can make your website load and work faster.

These next few sections discuss the process of including Business Casual on your web page.

Customizing Business Casual

If you just want to use Business Casual as is with its prebuilt business-casual.css and business-casual.min.js files, then you can skip this part. Otherwise, read on.

To make customization of Business Casual easy, I created a LESS file known as customize.less where you can tweak things like colors, gutter sizes, radii distances and the like in a single file. If you are really adventurous, you can even modify business-casual.js to make the framework's dynamic features behave a little differently. (To learn more about business-casual.js, see the developer page.)

Unfortunately, if you want to customize Business Casual, you will have to rebuild Business Casual.

Heads up!
Customizing Business Casual means you will have to rebuild Business Casual with Grunt, which requires you to install a few dependencies on your machine.

I can already hear your complaints and you're right. I will say, however, that Grunt is a super handy build automator for Node that takes care of LESS compilation and Javascript minification for you without having to explicitly install a LESS compiler and a Javascript minifier (like you did with build.sh). I'll walk you through the process of building Business Casual with Grunt:

Setup and Building

  1. Install Node. If you have Homebrew installed, run:

    $ brew install node
  2. Install grunt-cli using the Node Package Manager (npm) by running:

    $ npm install -g grunt-cli
  3. Install Business Casual's build dependencies by navigating to the root of your copy of Business Casual and running:

    $ npm install
  4. Rebuild Business Casual by navigating to the root of your copy of Business Casual and running:

    $ grunt
    By running grunt, Business Casual will attempt to be rebuilt. Grunt will let you know if the build succeeded or failed with errors. In the latter case, Grunt will show you what prevented the build from completing successfully so that you can fix these mistakes and rebuild again.

Linking & Bootstrapping

Linking Static Features

Linking business-casual.css to your HTML document will add Business Casual's static features:

<link rel="stylesheet" type="text/css" href="css/business-casual.css">

Bootstrapping Dynamic Features

It's a little more complicated to use Business Casual's dynamic features. The first step is to include business-casual.min.js to your HTML document:

<script type="text/javascript" src="js/business-casual.min.js"></script>
Heads up!
In v1.1, the deployment script is called business-casual-min.js. In v1.2, the deployment script is called business-casual.min.js.
Pro tip
Use business-casual.js for development, but use business-casual.min.js for deployment.

Once this is done, start a script tag in the head of your HTML document. Then, instantiate an instance of Business Casual and call the bootstrap function on it to get the framework to bootstrap:

window.onload = function() { var bc = new BusinessCasual(); bc.bootstrap(); }

Basics

Doctypes

You should always start your document with the HTML doctype.

<!DOCTYPE html> <html lang="en"> ... </html>

Containers

All of your content should fit inside a container div. This centers your content on the page and makes it interact nicely with other elements.

<div class="container"> ... </div>

Headings and Asides

Business Casual supports all six heading levels, as well as elements known as asides for the first three levels. Asides appear in a white, italic font to the right of the header.

<h1>Hello world<span class="aside">Oh hai</span></h1>

Separators

Separators can help make your page more logical. You can apply them to headers or use them separately.

<h1 class="separate">Bacon</h1> <div class="separator"></div>

Code

Are you a nerd that's using my framework? Then you've probably got some code to show off! Use the code element to use a monospace font.

<code> ... </code>

Flexors

Flexor elements allow you to divide your content into columns. The widths of the columns are assigned a factor between 1 and 12, such that the sum of the factors for the flexor is 12.

Usage

You must first make a flexor element, and then place columns inside of the flexor.

<div class="flexor"> <div class="column-#"> ... </div> </div>
Watch out!
The behavior of a flexor is undefined if the sum of the factors is more than 12!

Heroes

You can use heroes to welcome vistors to your page and to divide your page up logically.

Hero Units

An example of a hero unit is the one displayed at the top of this page. They require business-casual.js to function and can be used to welcome visitors to your webpage.

You can place an image inside of a hero unit using an img tag, as well as lay text over the image. Just be sure to use the hero-unit-text span around the text you want to display. Hero units center text within themselves by default.

<div class="hero-unit"> <span class="hero-unit-text"> Here is some text. </span> <img src="img.jpg" /> </div>

You can choose where you want the hero unit text to bias using the left and right classes.

... <span class="hero-unit-text left"> This text will bias left. </span> ...

For more effect, you can use the heroic class to make certain text stand out. This effect is actually used in the hero unit at the top of this page.

... <span class="hero-unit-text"> <span class="heroic">I'm freakin' awesome.</span> <br/> I will not display as awesomely. </span> ...

You can even mix other elements into your hero unit. In the following example, a button is used in the hero unit.

<div class="hero-unit"> <span class="hero-unit-text"> <span class="heroic">Hai</span> <br/> I can haz cheezburger? <br/> <a class="button">Yes, you may</a> </span> <img src="cat.jpg" /> </div>

Sidekick Units

Sidekick units are smaller than hero units and are used to divide your page into logical sections. There are few examples of sidekick units on this page. Again, you choose an image for the sidekick unit to display, as well as text that you might want over the image. Just be sure to use the sidekick-unit-text span around the text you want to display. And just like hero units, you can choose to which side of the sidekick unit that the text biaes. Text will center within the sidekick unit by default.

<div class="sidekick-unit"> <span class="sidekick-unit-text right"> Hello world! </span> <img src="img.jpg" /> </div>

Just like hero units, you can incorporate other elements into the text of your sidekick units. Keep in mind, however, that sidekick units are not as tall as hero units and that they do not implement the heroic span class in Business Casual.

Parallax

Adding parallax to any hero in Business Casual is easy; just add the parallax class.

<div class="sidekick-unit parallax"> <img src="parallax-me.jpg" /> </div>

Navigation Bars

Navigation bars are probably the most intricate elements of Business Casual and I am very proud to introduce them to you. While you can design your own navigation bars, Business Casual's navigation bars include a variety of features that can help make your page more dynamic and unique.

Static Features

To create a navigation bar in Business Casual, surround a ul element in a navigation-bar div. You can apply the center and right classes to the ul element to bias content; content is biased left by default. You can apply links to any element in the navigation bar, or even use an image as an element instead.

<div class="navigation-bar"> <!-- Content biases left by default --> <ul> <li>Kittens</li> <li><a href="https://google.com">Puppies</a></li> <li><img src="hamster.jpg" /></li> </ul> </div>

A logo is an element within a navigation bar that displays in a darker color and in bold text. These are good for displaying the name of your website.

<div class="navigation-bar"> <ul class="center"> <li class="logo">Architecture Portfolio</li> ... </ul> </div>

If you want a navgiation bar to follow the user around the page as they scroll, you can apply the magnetic class to the navigation bar.

<div class="navigation-bar magnetic"> ... </div>
Pro Tip
If you include business-casual.js, you can put multiple navigation bars on the page and Business Casual will intelligently stack them one after another, depending on their relative order on the page.

Dynamic Features

These features require business-casual.js in order to function.

Similar to the magnetic class is the sticky class. Similar to magnetic, sticky will cause a navigation bar to stick to the top of a browser's viewport as the user scrolls. The main difference, however, is that a sticky navigation bar will only fix itself to the top of the viewport when the user scrolls past the navigation bar; that is, the navigation bar will remain static on the page until the user scrolls past it and then stick itself to the top of the viewport.

<div class="navigation-bar sticky"> ... </div>

You may want a navigation bar to disappear after the user scrolls past a certain point on your page. You can accomplish this behavior with the fadeOut attribute. Specify the ID of an element on your page where you want the navigation bar to fade out after scrolling down past or fade in after scrolling up past. Navigation bars will automatically fix to their original position on the page if they are sticky.

<div class="navigation-bar magnetic" fadeOut="fader"> ... </div> ... <!-- This is the point where the navigation bar will fade --> <div id="fader"> ... </div>
Heads up!
Make sure that the ID of the element you choose for the navigation bar to fade out at appears below the navigation bar itself.
Pro tip
The fadeOut attribute only works with sticky navigation bars at this time. A patch will be released in the future to allow this feature to work with magnetic navigation bars.

Images & Glassboxes

Any good website has images on it. Business Casual provides you with three options when it comes to displaying images.

You can give an image rounded corners by using the rounded class, or a circular edge by using the circular class. You can add a border to an image by using the border class.

<img src="img.jpg" class="rounded" /> <img src="img.jpg" class="rounded border" /> <img src="img.jpg" class="circular" /> <img src="img.jpg" class="circular border" />

Including business-casual.js will allow you to create interactive slideshows known as glassboxes.

First, create a glassbox div. Then, for each image that you want, create a glassbox-slide that contains the picture you want to display. Optionally, you can also include a caption in the glassbox-slide using glassbox-caption. Finally, choose which glassbox-slide you want to display first by adding the glassbox-present class to it.

Be sure to include the glassbox-left-arrow and glassbox-right-arrow, or your vistors won't be able to use your glassbox.

<div class="glassbox"> <img class="glassbox-left-arrow" src="./arrow_left.png" /> <img class="glassbox-right-arrow" src="./arrow_right.png" /> <div class="glassbox-slide glassbox-present"> <span class="glassbox-caption"> The Mayans killed their greatest warriors on top there! </span> <img src="./scary.jpg" /> </div> ... </div>

You must provide your own image assets for the glassbox-left-arrow and glassbox-right-arrow buttons; these work pretty well.

Tables

Business Casual provides you with some basic table styling along with cell coloration. Use the rounded-table class to use the Business Casual style table. If you are planning on using a thead element in your table, you should also use the full class. This ensures that the table displays correctly.

<table class="rounded-table full"> <thead> <tr> <th>School</th> <th>Mascot</th> </tr> </thead> <tbody> <tr> <th>Duke</th> <th>Blue Devils</th> </tr> ... </tbody> </table>

<table class="rounded-table"> <tbody> <tr> <th class="left">Data</th> </tr> ... </tbody> </table>

Assets

Use a well when you want to wrap an element in a box to seperate it from other content. The code below is wrapped in a well.

<div class="well"> ... </div>

You can change links into buttons by using the button class on an a element. You can make a button appear smaller by using the small class, or larger by using the large class.

<a class="button small">Small</a> <a class="button">Button</a> <a class="button large">Large</a>
