Originally designed for my personal website, Business Casual is a super-minimalistic front-end framework that makes styling online portfolios and personal websites simple.

Business Casual is not responsive, nor is it designed for rich web applications. (If you want a good framework for that, go check out Bootstrap or Foundation.) What Business Casual excels at, however, is providing a simple, clean look to anything you want to show off online: your portfolio, your organizations's website, your company's homepage...

Stay impeccable.

About Business Casual

The latest version of Business Casual is v 1.0.1.

Features

Written in LESS

Business Casual's CSS was written completely in the LESS stylesheet langauge, making updates and customization a breeze.

No jQuery

Business Casual's Javascript has no jQuery dependencies in it, making for faster loading times. In fact, business-casual.js has no dependencies in it at all; only pure Javascript.

Built for the Future

Business Casual was designed for HTML 5 and CSS 3 compliant browsers, meaning that it is built with the latest and greatest web technology in mind.

Open Source

Business Casual is completely open source and is currently hosted on GitHub for the world to see. I don't care what you do with it, just remember to include my name somewhere! (See the MIT License further down for more information.)

Getting Started

In general, you can either use Business Casual as is or you can customize Business Casual to fit your own sense of style; whatever you do is completely up to you.

Heads up!
If you plan on customizing Business Casual, you should have a LESS compiler and a Bash shell on your machine. For more information, visit the LESS website.

Pulling

When you pull the Business Casual repository from GitHub, you should be provided with the following:

File Description
index.html This HTML document.
build.sh A shell script that builds Business Casual for you.
css/ A directory containing pre-built CSS that defines Business Casual.
js/ A directory containing Javascript that defines Business Casual's dynamic features.
less/ A directory containing LESS files that are used to build Business Casual. This is also where customize.less is located.
img/ A directory containing the images used on this document.
readme.md A markdown file for GitHub. Don't sweat this.

Customizing

If you do not plan on customizing Business Casual, skip this step.

In order to customize Business Casual, you will need the LESS compiler on your machine so you can compile from LESS to CSS. The LESS compiler is an executable called lessc. Before continuing, you need to ensure that lessc is installed on your machine and that you have lessc in your $PATH. (If you don't know if lessc is in your path, type which lessc at the Bash prompt.) For more information, visit the LESS website.

Now, open customize.less in your favorite text editor.

Careful!
Messing with other LESS files in the /less folder is strongly discouraged unless you are incredibly sure of what you're doing!

Inside, you will see a bunch of definitions for elements in Business Casual. You can modify these values to change the way Business Casual presents itself. When you are satisfied, simply execute the build.sh Bash script. If errors are produced while running build.sh, something may be wrong with your version of Business Casual. In this case, you will have to repull and rebuild Business Casual.

Another description on how to customize Business Casual can be found in readme.md.

Linking

Linking business-casual.css to an HTML document is fairly straightforward.

<link rel="stylesheet" type="text/css" href="css/business-casual.css">

If you need it, you can also add business-casual.js.

<script type="text/javascript" src="js/business-casual.js"></script>

After linking business-casual.js to your document, start it by instantiating Business Casual and then calling the go function on it.

window.onload = function() { var bc = new BusinessCasual(); bc.go(); }

Business Casual Documentation

Basics We need to start somewhere.

Doctypes

You should always start your document with the XHTML doctype.

<!DOCTYPE xhtml> <html lang="en"> ... </html>

Containers

All of your content should fit inside a container div. This centers your content on the page and makes it interact nicely with other elements.

<div class="container"> ... </div>

Headings and Asides

Business Casual supports all six heading levels, as well as elements known as asides for the first three levels. Asides appear in a white, italic font to the right of the header.

<h1>Hello world <span class="aside">Oh hai</span></h1>

Seperators

Seperators can help make your page more logical. You can apply them to headers or use them seperately.

<h1 class="seperate">Bacon</h1> <div class="seperator"></div>
Heads up!
For headers, use the class seperate, but for divs, use the class seperator!

Code

Are you a nerd that's using my framework? Then you've probably got some code to show off! Use the code element to use a monospace font.

<code> ... </code>

Flexors A simple gridding system.

Flexor elements allow you to divide your content into columns. The widths of the columns are assigned a factor between one and ten, such that the sum of the factors for the flexor is ten.

Usage

You must first make a flexor element, and then place columns inside of the flexor.

<div class="flexor"> <div class="column-#"> ... </div> </div>
Careful!
The behavior of a flexor is undefined if the sum of the factors is more than ten!
FYI
Flexors have a negative margin applied to them so that they align properly to the left edge of their parent. As a result, you may think that the flexors below do not take the width of their container. False! "Right-click, Inspect Element" the flexors below to see for yourself.

Heroes No superpowers required.

You can use heroes to welcome vistors to your page and to divide your page up logically.

Hero Units

An example of a hero unit is the one displayed at the top of this page. They require business-casual.js to function and can be used to welcome visitors to your webpage.

You can place an image inside of a hero unit using an img tag, as well as lay text over the image. Just be sure to use the hero-unit-text span around the text you want to display. Hero units center text within themselves by default.

<div class="hero-unit"> <span class="hero-unit-text"> Here is some text. </span> <img src="img.jpg" /> </div>

You can choose where you want the hero unit text to bias using the left and right classes.

... <span class="hero-unit-text left"> This text will bias left. </span> ...

For more effect, you can use the heroic class to make certain text stand out. This effect is actually used in the hero unit at the top of this page.

... <span class="hero-unit-text"> <span class="heroic">I'm freakin' awesome.</span> <br/> I will not display as awesomely. </span> ...

You can even mix other elements into your hero unit. In the following example, a button is used in the hero unit.

<div class="hero-unit"> <span class="hero-unit-text"> <span class="heroic">Hai</span> <br/> I can haz cheezburger? <br/> <a class="button">Yes, you may</a> </span> <img src="cat.jpg" /> </div>

Sidekick Units

Sidekick units are smaller than hero units and are used to divide your page into logical sections. There are few examples of sidekick units on this page. Again, you choose an image for the sidekick unit to display, as well as text that you might want over the image. Just be sure to use the sidekick-unit-text span around the text you want to display. And just like hero units, you can choose to which side of the sidekick unit that the text biaes. Text will center within the sidekick unit by default.

<div class="sidekick-unit"> <span class="sidekick-unit-text right"> Hello world! </span> <img src="img.jpg" /> </div>

Just like hero units, you can incorporate other elements into the text of your sidekick units. Keep in mind, however, that sidekick units are not as tall as hero units and that they do not implement the heroic span class in Business Casual.

Navigation Bars The pain is over.

Navigation bars are probably the most intricate elements of Business Casual and I am very proud to introduce them to you. While you can design your own navigation bars, Business Casual's navigation bars include a variety of features that can help make your page more dynamic and unique. They look great, too!

Static Features

To create a navigation bar in Business Casual, surround a ul element in a navigation-bar div. You can apply the center and right classes to the ul element to bias content; content is biased left by default. You can apply links to any element in the navigation bar, or even use an image as an element instead.

<div class="navigation-bar"> <!-- Content biases left by default --> <ul> <li>Kittens</li> <li><a href="https://google.com">Puppies</a></li> <li><img src="hamster.jpg" /></li> </ul> </div>

A logo is an element within a navigation bar that displays in a darker color and in bold text. These are good for displaying the name of your website.

<div class="navigation-bar"> <ul class="center"> <li class="logo">Architecture Portfolio</li> ... </ul> </div>

If you want a navgiation bar to follow the user around the page as they scroll, you can apply the magnetic class to the navigation bar.

<div class="navigation-bar magnetic"> ... </div>
FYI
If you include business-casual.js, you can put multiple magnetic navigation bars on the page and Business Casual will intelligently stack them one after another, depending on their relative ordering on the page.

Dynamic Features

These features require business-casual.js in order to function.

Similar to the magnetic class is the sticky class. Similar to magnetic, sticky will cause a navigation bar to stick to the top of a browser's viewport as the user scrolls. The main difference, however, is that a sticky navigation bar will only fix itself to the top of the viewport when the user scrolls past the navigation bar; that is, the navigation bar will remain static on the page until the user scrolls past it and then stick itself to the top of the viewport. (This is an incredibly cool feature!)

<div class="navigation-bar sticky"> ... </div>
FYI
Just like before, Business Casual is smart enough to stack sticky navigation bars one after another depending on their relative ordering on the page.

You may want a navigation bar to disappear after the user scrolls past a certain point on your page. You can accomplish this behavior with the fadeOut attribute. Specify the ID of an element on your page where you want the navigation bar to fade out after scrolling down past or fade in after scrolling up past. Navigation bars will automatically fix to their original position on the page if they are sticky.

<div class="navigation-bar magnetic" fadeOut="fader"> ... </div> ... <div id="fader"> ... </div>
Heads up!
Make sure that the ID of the element you choose for the navigation bar to fade out at appears below the navigation bar!

Images & Crossfaders Show me what you've got.

Any good website has images on it. Business Casual provides you with three options when it comes to displaying images.

You can give an image rounded corners by using the rounded class, or a circular edge by using the circular class. You can add a border to an image by using the border class.

<img src="img.jpg" class="rounded" /> <img src="img.jpg" class="rounded border" /> <img src="img.jpg" class="circular" /> <img src="img.jpg" class="circular border" />

If you include business-casual.js, you can create slideshows of images by making a crossfader. First, surround all the images you want to appear in the crossfader using the crossfader-frame class. Then, place img elements inside of the crossfader-frame, giving each the crossfader-image class. Finally, choose an image to kickoff the crossfader by using the crossfader-show class.

Make sure that the images are in the relative order you want them to display, starting with the crossfader-show image. Below is an example of how to use a crossfader with a sample.

<div class="crossfader-frame" style="width:250; height:250;"> <img src="img1.jpg" class="crossfader-image crossfader-show rounded" /> <img src="img2.jpg" class="crossfader-image rounded" /> <img src="img3.jpg" class="crossfader-image rounded" /> </div>
Careful!
Not applying a style attribute to the crossfader-frame could produce undefined behavior. For safety, make sure that you set the width and the height of the crossfader-frame!

Tables Data points are sexy.

Business Casual provides you with some basic table styling along with cell coloration. Use the rounded-table class to use the Business Casual style table. If you are planning on using a thead element in your table, you should also use the full class. This ensures that the table displays correctly.

<table class="rounded-table full"> <thead> <tr> <th>School</th> <th>Mascot</th> </tr> </thead> <tbody> <tr> <th>Duke</th> <th>Blue Devils</th> </tr> ... </tbody> </table>

Buttons What does this button do?

You can change links into buttons by using the button class on an a element. You can make a button appear smaller by using the small class, or larger by using the large class.

<a class="button small">Small</a> <a class="button">Button</a> <a class="button large">Large</a>

About Me Too much caffeine and not enough social skills.

I'm currently a senior at Virginia Tech studying Computer Science, graduating in May 2014. I needed an internship desperately and all the cool kids in my major that got internships with Google and Microsoft had a personal website. So I thought, "Why don't I make one?". And that's where it all began.

I wrote Business Casual to help update my personal website quickly and easily without having to remember too much. After a few friends asked if I had the CSS on GitHub (which I didn't), I decided to write a formal framework that anyone could use to build their own website to showcase their work in a beautiful way.

Many of the elements in Business Casual exist because I needed them in my own website. As such, it's likely that Business Casual will evolve and change over time. In reality, this whole thing is more of an experiment to see if I could write such a framework. However, if you have some requests for features that you'd like to see added to Business Casual, feel free to add an issue to the GitHub repository.
